# Interior Ergonomics — Справочник для проектирования жилых интерьеров

Источник данных: Neufert "Bauentwurfslehre", Panero & Zelnik "Human Dimension", DIN 33402-2.

## Базовые принципы

- Человек — основная единица измерения. Все размеры рассчитываются от параметров тела.
- Для зон досягаемости (reach) — использовать 5-й перцентиль (самые короткие руки).
- Для проходов и зазоров (clearance) — использовать 95-й перцентиль (самые крупные фигуры).
- Учитывать не только статические размеры тела, но и пространство для движений: наклонов, поворотов, вытягивания рук.

## Кухонный рабочий треугольник

Правила:
1. Три вершины: мойка — плита — холодильник
2. Сумма сторон: 3600–6000 мм (оптимально 5500–6000)
3. Каждая сторона: 1200–2700 мм
4. Мойка — плита: ≤ 1800 мм
5. Рабочий поток: столешница → плита → столешница → мойка → столешница (не прерывать дверями или высокими шкафами)
6. Сквозные пути движения НЕ должны пересекать рабочий треугольник

## Правила валидации планировки

При проверке сгенерированной расстановки мебели выполнить следующие проверки:

1. **Проходы** — все основные маршруты ≥ 800 мм, второстепенные ≥ 600 мм
2. **Кухонный треугольник** — сумма сторон 3600–6000, мойка-плита ≤ 1800
3. **Зазоры санузла** — перед унитазом ≥ 600, перед раковиной ≥ 550
4. **Вокруг кровати** — минимум 600 мм с трёх сторон (или двух, если одна у стены)
5. **Стулья за столом** — от спинки стула до стены/мебели ≥ 900 мм
6. **Двери** — не перекрываются мебелью, дуга открывания свободна
7. **Окна** — доступ к окнам не заблокирован (минимум 600 мм перед окном)
8. **Мебель не пересекается** — ни один объект не наложен на другой
9. **Мебель внутри комнаты** — все объекты полностью внутри периметра
10. **Розетки / выключатели** — мебель не блокирует доступ

Если правило нарушено — в JSON-ответ добавить:
```json
{
  "violations": [
    {
      "rule": "min_passage",
      "severity": "error",
      "message": "Проход между диваном и стеной = 450 мм (минимум 600)",
      "objects": ["sofa_1", "wall_north"],
      "actual_mm": 450,
      "required_mm": 600
    }
  ]
}
```

## Формат JSON для мебели в планировке

При генерации расстановки мебели использовать этот формат:
```json
{
  "room": {
    "width_mm": 5000,
    "length_mm": 4000,
    "type": "living_room",
    "doors": [{"wall": "south", "offset_mm": 1000, "width_mm": 900, "swing": "inward_left"}],
    "windows": [{"wall": "north", "offset_mm": 1500, "width_mm": 1500}]
  },
  "furniture": [
    {
      "id": "sofa_1",
      "type": "sofa_3seat",
      "width_mm": 2200,
      "depth_mm": 900,
      "x_mm": 1400,
      "y_mm": 3000,
      "rotation_deg": 0,
      "label": "Диван 3-местный"
    }
  ],
  "validation": {
    "passed": true,
    "violations": [],
    "score": 85
  }
}
```

## Скоринг планировки (0–100)

| Критерий | Вес | Описание |
|---|---|---|
| Проходы ≥ минимума | 30 | Все маршруты свободны |
| Функциональные зоны | 20 | Кухонный треугольник, зоны санузла |
| Свет от окон | 15 | Мебель не блокирует естественный свет |
| Пропорции | 15 | Мебель соразмерна комнате |
| Доступность дверей | 10 | Дуги дверей свободны |
| Розетки / выключатели | 10 | Доступ не заблокирован |

Score < 50 — планировка неприемлема, предложить альтернативу.
Score 50–70 — приемлемо, но есть замечания.
Score > 70 — хорошее решение.
