## Фаза 1: Профессиональный вид (исправить то, что уже есть)

### 1.1 Замена технических промптов на дизайнерские комментарии

**Проблема:** Под изображениями в Concept Board выводятся англоязычные промпты нейросети (`log house interior natural light`, `compact linen sofa durable fabric`). Документ выглядит как черновик нейросети.

**Решение:** Каждый комментарий — 3 элемента:
1. Что это (конкретный материал/мебель)
2. Зачем здесь (связь с эмоцией/функцией проекта)
3. Технический параметр (габариты, сегмент, эксплуатация)

**Задачи для Claude Code:**
- Создать шаблон генерации комментариев, который принимает данные из брифа (бюджет, табу, стиль, состав семьи, животные) и формирует осмысленный текст
- Добавить кнопку «Редактировать комментарии» для ручной правки
- Добавить возможность автоперегенерации на основе «Критериев успеха» из брифа

### 1.2 Цветовая палитра: чипы вместо фото

**Проблема:** В разделе палитры — интересные фотографии с промптами вместо профессиональных цветовых кодов.

**Решение:** UI-компонент «Цветовой чип».

**Задачи для Claude Code:**
- Создать компонент `ColorChip`: квадрат с заливкой (`background-color` по Hex) + строка с кодом (RAL/NCS) + роль (база/акцент)
- Реализовать извлечение доминирующих цветов из референсов (Vision AI → Hex)
- Добавить mapping-таблицу Hex → ближайший RAL Classic / NCS
- При генерации комментария к чипу — обосновывать выбор цвета на основе брифа
- Переименовать раздел: «Цветовая палитра» → «Палитра стандартов RAL / NCS»

### 1.3 Переименование интерфейса на профессиональный язык

**Проблема:** Кнопки и заголовки используют технический/разговорный язык вместо профессионального.

**Задачи для Claude Code** (полный список замен — см. PROJECT_CONTEXT.md, раздел 4):
- Заголовки разделов Concept Board: «Атмосфера / Найб» → «Визуальная атмосфера и эмоция», «Цветовая палитра» → «Палитра стандартов RAL / NCS», добавить «Карта материалов и текстур», «Габариты мебели и эргономика»
- Кнопки: «Новый проект» → «Загрузить исходники», «Перезапустить анализ» → «Выявить противоречия», «Пройти style narrowing» → «Синхронизировать вкус»
- Экспорт: «ЭКСПОРТ PDF» → «Экспорт приложения к договору» / «Скачать клиентскую версию»
- Подписи под вопросами: добавить «Разблокирует этап: [Название]»
- Под изображениями мебели: автоматически подставлять реальные габариты

### 1.4 Качество PDF-документа

**Проблема:** Текст в PDF — сырой текст с повторами, не пригоден для отправки клиенту.

**Задачи для Claude Code:**
- Переработать шаблон PDF-экспорта: чёткие разделы с заголовками, без технического мусора
- Добавить разделы: «Критерии успеха», «Табу», «Бюджет и ограничения»
- Зафиксировать точные технические параметры: верифицированные размеры помещений, габариты мебели
- Добавить блок согласования в конце: поле для клиента + поле для дизайнера + дата + подписи
- Реализовать версионность: футер с датой и статусом (`Draft concept, requires designer review` → `Утверждённая версия`)

## Фаза 2: Интеллектуальная логика

### 2.1 Детектор противоречий

**Задачи для Claude Code:**
- Создать модуль `ConflictDetector` — принимает данные брифа, возвращает список конфликтов
- Типы конфликтов для реализации:
  - Бюджет vs стоимость материалов (сумма > лимит)
  - Метраж из разных источников (план ≠ заметки) — критическое предупреждение
  - Стиль vs ограничения (противоречие между желаниями и табу)
  - Материалы vs функциональные требования (деликатные ткани + животные)
- Приоритизация: «Критично: Влияет на планировку» → «Важно: Влияет на бюджет» → «Опционально: Детализация»
- UI: блок «Конфликт данных» с вопросами, под каждым — «Разблокирует этап: [Название]»
- Генерация уточняющих вопросов для клиента на основе конфликта

### 2.2 Валидатор эргономики (MVP — 10 правил)

**Задачи для Claude Code:**
- Создать файл `ergonomics-rules.json` с 10 правилами (полный список с параметрами — PROJECT_CONTEXT.md, раздел 2.6):
  - Кухня: рабочий треугольник (120–270 см), мойка+посудомойка, безопасность плиты
  - Проходы: минимум 60 см, точность до 2 см, масштабирование для малых пространств
  - Зонирование: сценарии света, кровать не у окна
  - Хранение: по числу проживающих, встройка в сложную геометрию
- Создать модуль `ErgonomicsValidator`: принимает планировку → прогоняет через правила → список предупреждений
- Реализовать верификацию обмерочного плана: сверка размеров из разных источников, предупреждение при расхождении
- Учёт зон открывания дверей и шкафов
- Добавить кнопку «Проверить проходы (Валидатор)» в интерфейс
- Предусмотреть кастомизацию: дизайнер загружает свои правила через профиль

### 2.3 Автоматический расчёт бюджета

**Задачи для Claude Code:**
- Создать модуль `BudgetCalculator` с алгоритмом:
  1. Категоризация материалов по сегментам (эконом / средний / премиум) — каждый материал в КБ имеет привязку
  2. Извлечение площадей зон из обмерочного плана
  3. Расчёт: `(площадь зоны) × (средняя цена материала в сегменте)`
  4. Учёт функциональных требований: если «прочное покрытие» — подставить кварцвинил, не ламинат
  5. Сверка общей суммы с лимитом клиента — в реальном времени
  6. При превышении — предупреждение + уточняющий вопрос
- Реализовать динамический пересчёт: при замене материала/мебели — мгновенное обновление сметы
- UI-блок с итоговой суммой и индикатором (в пределах / превышение)
- Передача сегмента в управляемый промпт

## Фаза 3: Персонализация

### 3.1 Профиль дизайнера

**Задачи для Claude Code:**
- Реализовать экран `DesignerProfile` по согласованному UX:
  - 5-осевые слайдеры визуального языка
  - Persistent текстовое поле описания стиля
  - Drag-and-drop загрузка портфолио с AI-анализом
  - Поле внешних ссылок
- Модуль «Мои стандарты»: интерфейс для задания базовых параметров работы ИИ
- «Личные табу» дизайнера: список жёстких исключений, которые ИИ применяет автоматически
- Кастомизация правил эргономики: загрузка своих чек-листов

### 3.2 Синхронизация «Дизайнер × Клиент»

**Задачи для Claude Code:**
- Компонент `StyleSyncDashboard`:
  - Блок «Согласовано» — элементы, одобренные дизайнером и совпадающие с вкусом клиента
  - Блок «На обсуждение» — конфликтующие точки с уточняющими вопросами
- Фильтрация «свой/чужой» на входе: совпадает ли стиль дизайнера с ожиданиями клиента
- Боковая панель «Опора на бриф» — всегда видны текущие ограничения

### 3.3 Style Narrowing (Сужение стиля)

**Задачи для Claude Code:**
- Пошаговый мастер с карточным выбором (игровой UX):
  - Шаг «Цвет»: выбор палитры
  - Шаг «Материалы»: дерево, камень, текстиль
  - Шаг «Стиль»: минимализм, арт-деко, japandi и т.д.
- Группировка через эмоции (пример: «спокойный/уютный» вместо «japandi»)
- Во время выбора — боковая панель с табу, чтобы выбор не противоречил ограничениям

## Фаза 4: Продвинутые фичи

### 4.1 Управляемый промпт для визуализации
- Автосборка промпта из утверждённого концепта
- 6 обязательных параметров: габариты, материалы, RAL/NCS коды, стилистика, сегмент, негативные указания
- Кнопка «Подготовить промпт для рендера»

### 4.2 Связь цвета с базой мебели
- Атрибуты RAL/NCS у каждого предмета в КБ
- Многоуровневая фильтрация: Цвет + Бюджет + Габариты + Табу
- Автоматическая блокировка цветов из стоп-листа
- Кнопка «Передать в закупку»

### 4.3 Игровой UX для компоновки
- Клиент «играет» с вариантами расположения мебели
- Под каждый вариант ИИ мгновенно пересчитывает концепт и смету
- Проверка на вписываемость в план при каждом перемещении

### 4.4 Юридическая фиксация (полный цикл)
- Кнопки: «Зафиксировать для договора», «Утвердить состав проекта», «Завершить согласование (ТЗ)»
- Автозамена статуса в PDF: Draft → Утверждённая версия (Приложение №1)
- Блокировка редактирования после утверждения

## Как работать с Claude Code по этому плану

1. **Перед сессией** — скажи Claude Code: «Прочитай PROJECT_CONTEXT.md и ROADMAP_v2.md — это контекст проекта»
2. **Одна задача за сессию** — бери конкретный пункт (например, «1.2 Цветовая палитра: создай компонент ColorChip»), не пытайся сделать всё сразу
3. **Формулируй задачу конкретно** — «Создай компонент ColorChip, который принимает hex, название RAL и роль (база/акцент), и отображает квадрат с заливкой и подписью»
4. **Проверяй результат** перед переходом к следующей задаче
5. **Фиксируй прогресс** — после каждой выполненной задачи отмечай в этом файле (например, ✅)
