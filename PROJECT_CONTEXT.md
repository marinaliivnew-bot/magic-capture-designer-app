## 1. Что такое Magic Capture

Интеллектуальный инструмент для дизайнеров интерьера. Превращает «сырой» ввод от клиента (переписки, аудио, фото, Pinterest) в структурированный бриф и визуальную концепцию (Concept Board), пригодные для подписания как приложение к договору.

**Ключевая метафора:** «Архитектор брифа» — не черновик нейросети, а профессиональный документ.

**Принцип:** Дизайнер — редактор, а не «печатная машинка». ИИ делает черновую работу, дизайнер принимает решения.

**North Star Metric:** Time-to-Approved-Concept (TTAC) — время от первого входа до «концепт утверждён».

## 3. Модель данных (Supabase / PostgreSQL)

```sql
-- projects
id uuid PRIMARY KEY, session_id text NOT NULL,
name text, room_type text, dimensions_text text,
raw_input text, constraints jsonb, created_at timestamptz

-- briefs
project_id uuid REFERENCES projects(id) ON DELETE CASCADE,
users text, scenarios text, zones text, storage text,
style_likes text, style_dislikes text, constraints_practical text,
success_criteria text, completeness_score int

-- issues (contradictions + gaps)
project_id uuid, type text ('contradiction'|'gap'),
title text, evidence text, impact text, suggestion text

-- questions
project_id uuid, priority text, text text, unlocks text,
asked bool DEFAULT false, answer text

-- board_blocks
project_id uuid, block_type text, caption text, sort_order int

-- board_images
block_id uuid, url text, source_type text, source_url text,
attribution text, note text

-- rooms (отдельные помещения внутри проекта)
id uuid PRIMARY KEY, project_id uuid REFERENCES projects(id) ON DELETE CASCADE,
name text NOT NULL, room_type text DEFAULT 'other',
dimensions_text text, plan_url text, sort_order int, created_at timestamptz

-- designer_profile
id uuid PRIMARY KEY, session_id text NOT NULL,
style_description text, style_refs jsonb DEFAULT '[]',
hard_constraints jsonb DEFAULT '{}',  -- сюда же кладутся слайдеры визуального языка и пути загруженных файлов
ergonomics_rules jsonb DEFAULT '{}', custom_ergonomics_text text,
created_at timestamptz, updated_at timestamptz
```

ON DELETE CASCADE на всех дочерних таблицах → кнопка «Удалить проект» работает без доп. логики.

**Безопасность:** Row Level Security (RLS) включён на всех 8 таблицах выше, доступ ограничен по `session_id` через заголовок `x-session-id`. Полную и актуальную картину политик и истории миграций смотреть в `ADR.md` и напрямую в `supabase/migrations/` — список миграций растёт быстро, фиксировать конкретное число здесь нет смысла.

## 4. Функциональные модули

### 4.1 Мультимодальный ввод и анализ

**Форматы загрузки:** переписки, транскрипты аудио, заметки, Pinterest-ссылки, обмерочные планы (2D/3D), фотографии объекта.

**Автоматизация:** ИИ распределяет информацию по блокам (свет, стиль, материалы, бюджет). Кнопка «Заполнить габариты из текста» — AI парсит raw input и заполняет список комнат.

### 4.2 Детектор противоречий (ConflictDetector)

4 типа конфликтов:
- Бюджет vs стоимость материалов (сумма > лимит)
- Метраж из разных источников (план ≠ заметки) — критическое предупреждение
- Стиль vs ограничения (желания vs табу)
- Материалы vs функциональные требования (деликатные ткани + животные)

Приоритизация: «Критично: Влияет на планировку» → «Важно: Влияет на бюджет» → «Опционально: Детализация». Под каждым вопросом — «Разблокирует этап: [Название]».

### 4.3 Style Narrowing (Синхронизация вкуса)

Визуальный выбор стиля через карточки. Кураторские запросы к Unsplash (только интерьеры комнат, без дублей). Рандомизация из top-10 результатов. Аннотации к референсам («что именно нравится»).

### 4.4 Concept Board

5 обязательных разделов:
- **«Визуальная атмосфера и эмоция»** — 2–3 фото, задающих эмоцию
- **«Палитра стандартов RAL / NCS»** — цветовые чипы (ColorChip) с кодами Hex/RAL/NCS + роль (база/акцент)
- **«Карта материалов и текстур»** — конкретные покрытия, привязка к сегменту и функциональности
- **«Габариты мебели и эргономика»** — мебель с учётом реальных размеров помещения
- **«Освещение»** — сценарии света

Дизайнерские комментарии (3 компонента): Что это → Зачем здесь → Тех. параметр. Данные комнат, размеры, ответы клиента на уточняющие вопросы и результат Style Narrowing передаются в `generate-board`; борд структурируется по помещениям.

### 4.5 Валидатор эргономики

10 правил (кухонный треугольник, проходы ≥60 см, точность до 2 см, масштабирование мебели, сценарии света, спальное место не у окна, хранение по числу проживающих, встройка в сложную геометрию). JSON-база правил. Кнопка «Проверить проходы (Валидатор)».

### 4.6 Расчёт бюджета (BudgetCalculator)

Алгоритм: категоризация материалов по сегментам → извлечение площадей из плана → расчёт (площадь × цена сегмента) → сверка с лимитом → предупреждение при превышении. Динамический пересчёт при замене материала/мебели.

> **Примечание:** разделы 4.4 (габариты мебели), 4.5 (валидатор) и 4.6 (бюджет) реализованы как единый экран `FurniturePlanPage`: каталог мебели по ценовым сегментам + RAL-палитра цветов + валидация проходов + расчёт бюджета + модуль закупки (`ProcurementModal`).

### 4.7 PDF-экспорт

Разделы: бриф (8 секций), критерии успеха, табу, бюджет и ограничения, concept board с подписями, блок согласования (клиент + дизайнер + дата + подпись). Версионность: Draft → Готов к согласованию → Утверждённая версия.

### 4.8 Профиль дизайнера

Дизайнер задаёт: имя, визуальный язык через 5 слайдеров (Холодно↔Тепло, Строго↔Свободно, Просто↔Фактурно, Монохром↔Цветно, Классика↔Авангард), стандарты/ограничения (свободный текст), загружает файлы в две зоны — «Портфолио» и «База знаний» (PDF, изображения). Кнопка AI-анализа читает содержимое файлов (включая vision-анализ PDF, не просто имена файлов) и формирует профиль автоматически.

### 4.9 Дашборд проектов

Список всех проектов дизайнера с показателями: completeness_score, статус согласования (Draft / Locked / Approved), согласованные элементы, открытые конфликты по стилю, наличие готового борда. Режимы отображения: список / галерея, фильтр по статусу, закрепление важных проектов, шаринг, удаление.

### 4.10 Публичная ссылка на проект

Read-only страница для клиента по публичной ссылке — бриф и статус согласования (Draft / Locked / Approved), без прав на редактирование. Способ отправить проект клиенту на просмотр без захода в личный кабинет дизайнера.

## 6. Полный Workflow

```
1. ЗАГРУЗКА ИСХОДНИКОВ
   Переписки, аудио, фото, планы, Pinterest
        ↓
2. АВТОМАТИЧЕСКИЙ АНАЛИЗ
   ИИ распределяет по блокам + верификация обмерочного плана
        ↓
3. ДЕТЕКТОР ПРОТИВОРЕЧИЙ
   Конфликты → Уточняющие вопросы → Приоритизация
        ↓
4. СИНХРОНИЗАЦИЯ ВКУСА
   Анализ референсов → Style Narrowing → Дашборд совпадений
        ↓
5. CONCEPT BOARD
   Палитра RAL/NCS → Материалы → Мебель с габаритами
   Дизайнерские комментарии (Что → Зачем → Тех.параметр)
        ↓
6. ВАЛИДАЦИЯ
   Эргономика (10 правил) + Расчёт бюджета
        ↓
7. УПРАВЛЯЕМЫЙ ПРОМПТ
   Автосборка → Визуализация (Nano Banana / Midjourney)
        ↓
8. ЮРИДИЧЕСКАЯ ФИКСАЦИЯ
   PDF → Согласование → Приложение к договору
```

## 7. Планировочные принципы (из интервью с дизайнерами)

- **Иерархия помещений:** парадные зоны → больше света и пространства; частные → компактнее
- **Осевое проектирование:** оси движения «на свет», анфиладный принцип
- **Вид из окна** — важнейший фактор при расстановке мебели
- **Планировка — чёрно-белые чертежи:** пропорции и функциональность важнее картинки
- **Планы освещения и электрики** — после утверждения расстановки мебели
- **Десктопная версия** — приоритет; быстрый экспорт в PDF
