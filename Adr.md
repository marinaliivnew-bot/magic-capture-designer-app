# Architecture Decision Log (ADR)

> Зачем этот файл: чтобы Claude Code не предлагал альтернативы, которые уже были рассмотрены и отвергнуты.
> Формат: одно решение — один блок. Дата, выбор, почему, что отвергнуто.

---

## ADR-001: Supabase как единый бэкенд
**Дата:** октябрь 2025 (старт MVP)

**Выбор:** Supabase (PostgreSQL + Edge Functions + Storage + Auth в будущем)

**Почему:**
- Бесплатный tier покрывает MVP (500 MB, 2 GB трафика)
- Единое место: БД + серверные функции + файловое хранилище
- Edge Functions решают проблему секретных ключей (Anthropic API key не утекает на фронт)
- CLI позволяет деплоить схему и функции одной командой
- При росте легко добавить Supabase Auth поверх session_id

**Отвергнуто:**
- Firebase — менее удобен для реляционных данных, SQL-запросы сложнее
- Прямые вызовы Anthropic API с фронта — утечка API key
- NestJS + Redis + BullMQ — избыточно для соло-разработчика, обсуждалось и вынесено за скобки

## ADR-002: Cloudflare Pages (деплой)
**Дата:** апрель 2026

**Выбор:** Cloudflare Pages с GitHub autodeploy

**Почему:**
- Бесплатный tier без ограничений по трафику
- Автодеплой при push в main — не нужен CI/CD pipeline
- Глобальный CDN из коробки

**Отвергнуто:**
- Netlify — использовался изначально, но Cloudflare Pages проще по конфигурации и не имеет лимита на bandwidth в бесплатном плане
- Vercel — требует привязку к Next.js экосистеме, у нас чистый React + Vite

## ADR-003: Anthropic API (Claude Sonnet) для генерации
**Дата:** октябрь 2025

**Выбор:** Claude Sonnet через Supabase Edge Functions

**Почему:**
- Лучшее качество структурированного вывода (JSON) среди доступных моделей
- Русский язык на уровне, достаточном для интерфейса дизайнера
- max_tokens: 4096 хватает для брифа из 8 секций + вопросов + борда
- Парсинг: строго JSON без markdown-обёртки, retry 1 раз при ошибке

**Отвергнуто:**
- GPT-4 — тестировался, хуже следует инструкциям по формату JSON
- Локальные модели — нет смысла для облачного SaaS
- Gemini — не тестировался, нет причин переключаться

## ADR-004: Unsplash API как основной источник изображений
**Дата:** октябрь 2025

**Выбор:** Unsplash API → Together AI FLUX (fallback) → заглушка

**Почему:**
- Бесплатный tier (50 req/hour) покрывает MVP
- Качественные интерьерные фото с атрибуцией
- Не требует генерации — быстрее, чем AI-картинки
- Кэширование результатов в Supabase Storage снижает нагрузку на лимит

**Отвергнуто:**
- Pexels API — меньше интерьерного контента
- Только AI-генерация (Midjourney/FLUX) — медленно, дорого, нестабильное качество для интерьеров
- Pinterest API — нет публичного API для поиска

**Известные проблемы:**
- Кураторские запросы критичны — общие запросы вроде «minimalism interior» дают мусор
- Решено: файл `style-cards.ts` с ручными запросами + рандомизация из top-10

## ADR-005: React + Tailwind + Vite (фронтенд)
**Дата:** октябрь 2025

**Выбор:** React + Tailwind CSS, сборка через Vite

**Почему:**
- Lovable (первый конструктор, где начинался MVP) работает на React + Tailwind
- Vite — быстрая сборка, HMR, нативная поддержка TypeScript
- Tailwind — быстрая стилизация без написания CSS-файлов, responsive из коробки

**Отвергнуто:**
- Next.js — SSR не нужен для инструмента дизайнера, лишняя сложность
- Vue / Svelte — нет причин переключаться, экосистема скиллов заточена под React

## ADR-006: Edge Functions вместо клиентских вызовов API
**Дата:** март 2026

**Выбор:** Вся AI-логика (analyze-brief, generate-board) в Supabase Edge Functions

**Почему:**
- Anthropic API key хранится на сервере, не утекает в браузер
- CORS решается на уровне Edge Function (был баг с 404 + CORS — исправлен)
- Можно добавить rate limiting и логирование на серверной стороне
- Формат: `npx supabase functions deploy <name> --project-ref iyqfstkhpsalqxzhousy`

**Отвергнуто:**
- Прямые вызовы с фронта — утечка ключа
- Отдельный Node.js сервер — лишняя инфраструктура для соло-разработки

## ADR-007: PDF — текст + ссылки (без встроенных изображений)
**Дата:** октябрь 2025

**Выбор:** react-pdf / @react-pdf/renderer, PDF содержит текст + кликабельные ссылки на картинки

**Почему:**
- Встраивание изображений в PDF из браузера технически ненадёжно (CORS, размер файла, время генерации)
- Текстовый PDF генерируется мгновенно
- Для MVP достаточно — дизайнер может открыть ссылки

**Отвергнуто:**
- Встроенные изображения — отложено на будущее (блок 6.5 Build-Ready Export)
- Серверная генерация PDF (Puppeteer) — лишняя инфраструктура

## ADR-008: Анонимная авторизация через session_id
**Дата:** октябрь 2025

**Выбор:** Все проекты привязаны к session_id в localStorage, без регистрации

**Почему:**
- MVP без трения — дизайнер сразу начинает работу
- Кнопка «Удалить проект» каскадно удаляет всё (ON DELETE CASCADE)
- При росте → добавить Supabase Auth поверх, привязать session_id к user_id

**Отвергнуто:**
- Auth с самого начала — лишний барьер для MVP, снижает конверсию в первый проект

## ADR-009: VS Code + Claude Code (среда разработки)
**Дата:** май 2026

**Выбор:** VS Code + расширение Claude Code (ранее — Windsurf + Kimi K2.5)

**Почему:**
- Claude Code лучше следует контексту из CLAUDE.md и PROJECT_CONTEXT.md
- Скиллы (.claude/skills/) ускоряют на 30–40% и стабилизируют качество
- Единая экосистема: Claude для планирования (этот чат) + Claude Code для реализации

**Отвергнуто:**
- Windsurf + Kimi K2.5 — использовался ранее, но Cascade (агент Windsurf) удалял компоненты, которые не просили трогать
- Cursor — не тестировался, нет причин переключаться с рабочего сетапа

**Урок:** Всегда проверять структуру кода перед промптом в Claude Code — указывать конкретные файлы, а не «исправь страницу»

## ADR-010: Скиллы как способ управления качеством AI-кода
**Дата:** 7 мая 2026

**Выбор:** SKILL.md файлы в `.claude/skills/` — инструкции, которые Claude Code читает перед выполнением задачи

**Установлены:**
- `frontend-design` (Anthropic) — профессиональный UI
- `canvas-design` (Anthropic) — визуальные артефакты
- `interior-ergonomics` (кастомный) — 10 правил эргономики из Нойферта, Panero & Zelnik, DIN 33402-2
- `skill-creator` (Anthropic) — гайд по созданию новых скиллов

**Почему:**
- Без скилла Claude Code генерирует «generic AI UI» — с ним выдаёт профессиональный результат
- interior-ergonomics содержит точные размеры и правила — без него AI выдумывает цифры

**Источник:** `git clone https://github.com/anthropics/skills.git` + NotebookLM (экспертные заметки)

## ADR-011: Env-переменные — именование
**Дата:** март 2026

**Выбор:** `VITE_SUPABASE_ANON_KEY` (не `VITE_SUPABASE_PUBLISHABLE_KEY`)

**Почему:** Vite требует префикс `VITE_` для переменных, доступных на фронте. Supabase документация использует `anon key`. Смешение названий приводило к багу — ключ не подхватывался.

**Правило:** Все env-переменные: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_ANTHROPIC_KEY`, `VITE_UNSPLASH_KEY`

## ADR-012: Supabase Policies — паттерн пересоздания
**Дата:** апрель 2026

**Выбор:** Всегда `DROP POLICY IF EXISTS` перед `CREATE POLICY`

**Почему:** При повторном запуске миграции Supabase выдаёт ошибку «policy already exists». Паттерн DROP + CREATE решает это без ручного вмешательства.
